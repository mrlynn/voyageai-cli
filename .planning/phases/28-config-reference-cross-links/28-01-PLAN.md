<!-- Environment variables found in codebase (via grep) -->
Core / credentials:
  VOYAGE_API_KEY — Voyage AI API key (overrides vai config set api-key)
  VOYAGE_API_BASE — Override Voyage API base URL (advanced)
  MONGODB_URI — MongoDB connection string (overrides vai config set mongodb-uri)

LLM configuration:
  VAI_LLM_PROVIDER — LLM provider: anthropic, openai, ollama (overrides --llm-provider)
  VAI_LLM_MODEL — LLM model name (overrides --llm-model)
  VAI_LLM_API_KEY — LLM API key (overrides --llm-api-key)
  VAI_LLM_BASE_URL — LLM base URL for Ollama (overrides --llm-base-url)

Project defaults:
  VAI_DEFAULT_DB — Default MongoDB database (overrides .vai.json db)
  VAI_DEFAULT_COLLECTION — Default MongoDB collection (overrides .vai.json collection)
  VAI_DB — Alias for VAI_DEFAULT_DB (used in purge/refresh)
  VAI_COLLECTION — Alias for VAI_DEFAULT_COLLECTION (used in purge/refresh)
  VAI_CONFIG_PATH — Override config file path (default: ~/.vai/config.json)

Display / UX:
  VAI_NO_WELCOME — Set to 1 or true to suppress the welcome/onboarding flow
  VAI_MASK — Set to 1 or true to mask API keys in ping/doctor output

MCP server:
  VAI_MCP_SERVER_KEY — Auth key for MCP server endpoint
  VAI_MCP_VERBOSE — Set to 1 to enable verbose MCP server logging

Playground:
  VAI_PLAYGROUND_PASSWORD — Password for playground web UI (empty = no auth)

Telemetry / observability:
  VAI_TELEMETRY — Set to 0 or false to disable anonymous telemetry
  VAI_TELEMETRY_DEBUG — Set to 1 to print telemetry payloads to stderr
  DO_NOT_TRACK — Standard DNT flag; disables telemetry when set to 1 or true

Standard / inherited:
  DEBUG — Enables debug stack traces in several commands
  NODE_ENV — Standard Node.js environment (production suppresses some error details)

<!-- .vai.json chat block fields (from src/commands/chat.js chatConf usage) -->
Root-level fields (established in prior docs):
  version — Schema version, always 1
  model — Default embedding model for ingest/embed commands
  db — Default MongoDB database
  collection — Default MongoDB collection
  field — Embedding field name in documents
  inputType — Input type for embedding: document or query
  dimensions — Embedding dimensions (default 1024)
  index — Vector index name
  chunk.strategy — Chunking strategy: recursive, sentence, paragraph
  chunk.size — Chunk size in tokens
  chunk.overlap — Chunk overlap in tokens

chat block fields (v1.5 additions):
  chat.memoryStrategy — Default memory strategy: sliding_window, summarization, hierarchical
  chat.maxContextDocs — Max documents retrieved per turn (default: 5)
  chat.maxConversationTurns — Max turns before truncation (default: 20)
  chat.embeddingModel — Default embedding model for chat retrieval
  chat.systemPrompt — Override default system prompt
  chat.mode — Chat mode: pipeline or agent (default: pipeline)
  chat.showSources — Show retrieved sources after each response (default: true)
  chat.showToolCalls — Show tool call details in agent mode (default: true)

# .vai.json Schema

Opening paragraph: `.vai.json` is an optional project-level configuration file placed in your project root. It sets defaults for embedding model, MongoDB database and collection, chunking, and (since v1.5) chat behavior. CLI flags always override .vai.json values.

## Overview

```json
  "db": "mydb",

  "field": "embedding",
  "inputType": "document",
  "dimensions": 1024,
  "index": "vector_index",
  "chunk": {
    "strategy": "recursive",
    "size": 512,
    "overlap": 50
  },
  "chat": {
    "memoryStrategy": "sliding_window",
    "maxContextDocs": 5,

    "showToolCalls": true
  }
}
```
