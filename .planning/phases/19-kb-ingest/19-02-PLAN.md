**PDF text extraction:**

Add a `extractTextFromPDF(buffer)` function at module level. Use the `pdf-parse` npm package (or if not available, implement a lightweight approach). Check if `pdf-parse` is in package.json first:
- If available: `const pdfParse = require('pdf-parse'); const data = await pdfParse(buffer); return data.text;`
- If NOT available: Install it via `npm install pdf-parse` (add to plan verification). This is a well-established package with zero native dependencies -- pure JS PDF parser.

In the existing `POST /api/rag/ingest` handler, after reading each file:
1. Check if the file extension is `.pdf` -- if so, read the file as a Buffer (`fs.readFileSync(file.path)`) and call `extractTextFromPDF(buffer)` to get the text content
2. For non-PDF files, continue reading as utf8 text (existing behavior)

**Stage-level progress:**

Refactor the existing file ingest progress to emit stage-specific events. Currently it emits one progress event per file. Change to:

For each file:
1. `{type:'progress', stage:'reading', file: name, fileIndex: i, fileCount: total}`
2. After chunking: `{type:'progress', stage:'chunking', file: name, chunks: chunkCount, fileIndex: i, fileCount: total}`
3. For each chunk being embedded: `{type:'progress', stage:'embedding', file: name, current: chunkIdx+1, total: chunkCount, fileIndex: i, fileCount: total}`
4. After all chunks stored: `{type:'progress', stage:'storing', file: name, fileIndex: i, fileCount: total}`

After all files: `{type:'complete', kbName, docCount, chunkCount}` (unchanged)

Also update the multipart parser to handle binary PDF content correctly: change `const bodyStr = body.toString()` to handle binary boundaries properly. For PDF files, write the raw Buffer slice (not string) to the temp file. The simplest fix: after extracting parts by boundary, for file parts check if filename ends in `.pdf` -- if so, extract the content portion as a Buffer slice from the original `body` Buffer (not from the string conversion). Use `body.indexOf()` with Buffer boundaries to find content positions.

Actually, a simpler approach: since the multipart parser already writes files to tempDir, just change the file reading logic. When processing each file in the ingest loop, check extension:
- `.pdf`: `const buffer = fs.readFileSync(file.path); content = await extractTextFromPDF(buffer);`
- Others: `const content = fs.readFileSync(file.path, 'utf8');` (existing)

The binary parsing issue with the multipart boundary is that `body.toString()` may corrupt binary PDF data. Fix: store `body` as Buffer, find boundaries using Buffer operations. Use `body.indexOf('--' + boundary)` on the Buffer to split parts. For each part, find the `\r\n\r\n` separator, and write the content portion as raw Buffer to tempDir. This preserves PDF binary data.

Alternatively (simpler): use the `busboy` or `formidable` npm package for multipart parsing if not already available. BUT to minimize new dependencies, keep the manual parser but switch to Buffer-based boundary splitting:

```javascript
// Replace string-based parsing with Buffer-based
const boundaryBuf = Buffer.from('--' + boundary);
// Find boundary positions in body Buffer
// Extract file content as Buffer slices
// Write to tempDir with fs.writeFileSync(filepath, contentBuffer)
```

This ensures PDF binary data is preserved when written to the temp file.

2. In `ingestFiles()` (and any future `ingestText()`/`ingestURL()` from plan 19-01), update the progress event handler to show stage-level labels:

   Replace the simple progress handler with stage-aware rendering:
   ```javascript
   if (event.type === 'progress') {
     let label = '';
     let percent = 0;

     if (event.stage === 'reading') {
       label = `Reading ${event.file}...`;
       percent = 5; // small initial progress
     } else if (event.stage === 'chunking') {
       label = `Chunking ${event.file} (${event.chunks} chunks)`;
       percent = 15;
     } else if (event.stage === 'embedding') {
       label = `Embedding ${event.current}/${event.total} chunks`;
       percent = 15 + Math.round((event.current / event.total) * 70);
     } else if (event.stage === 'storing') {
       label = `Storing ${event.file}...`;
       percent = 90;
     } else {
       // Fallback for old-style progress (file-level)
       label = `${event.current}/${event.total} — ${event.file}`;
       percent = Math.round((event.current / event.total) * 100);
     }

     if (progressBar) progressBar.style.width = `${percent}%`;
     if (progressLabel) progressLabel.textContent = label;
   }
   ```

   Change: `input.accept = '.txt,.md';` to `input.accept = '.txt,.md,.pdf';`
