Add three new explain topics to the `concepts` object in explanations.js, placed after the existing `harness` topic:

1. **`sessions`** topic:
   - Title: "Chat Sessions"
   - Summary: "How to start, resume, list, and manage chat sessions"
   - Content must cover:
     - Starting a new session (automatic when MongoDB available)
     - Resuming with `--session <id>` (shows turn count)
     - Listing with `vai chat --list` (shows recent sessions with status, turn count, date)
     - `--all` flag to include archived sessions
     - Session lifecycle: ACTIVE -> PAUSED -> ARCHIVED
     - MongoDB fallback: sessions run in-memory when MongoDB unavailable
     - Session ID format (first 8 chars shown in prompts)
   - Links: ['https://docs.voyageai.com/']
   - tryIt: ['vai chat --list', 'vai chat --session <id>', 'vai chat --list --all']

2. **`memory-strategies`** topic:
   - Title: "Memory Strategies"
   - Summary: "How sliding window, summarization, and hierarchical memory work and when to use each"
   - Content must cover:
     - What memory management solves (fitting conversation history into LLM context window)
     - Token budget allocation: system prompt + context docs + current message + response reserve = history budget
     - **sliding_window** (default): Keeps most recent turns that fit. Fastest, no LLM calls. Best for short conversations or when recent context is sufficient.
     - **summarization**: Compresses older turns into LLM-generated summaries. 60/40 split (recent turns kept verbatim, older turns summarized). Best for long conversations where early context matters.
     - **hierarchical**: Combines recent turns + summaries + cross-session recall from past conversations. Best when working across multiple sessions on related topics.
     - How to select: `--memory-strategy <name>` flag or `/memory` slash command during chat
     - `/memory` command shows current strategy, token budget breakdown, and utilization
   - Links: ['https://docs.voyageai.com/']
   - tryIt: ['vai chat --memory-strategy sliding_window', 'vai chat --memory-strategy summarization', 'vai chat --memory-strategy hierarchical', 'vai chat   (then type /memory)']

3. **`cross-session-recall`** topic:
   - Title: "Cross-Session Recall"
   - Summary: "How past conversations inform new sessions via Voyage AI asymmetric embedding"
   - Content must cover:
     - What cross-session recall does: surfaces relevant context from past conversations when starting or resuming sessions
     - How it works: archived sessions generate summaries that are embedded using Voyage AI asymmetric embedding
     - Asymmetric approach: voyage-4-large for indexing summaries (accuracy), voyage-4-lite for queries (speed/cost)
     - When it activates: automatically with hierarchical memory strategy
     - The embedding space compatibility: all voyage-4 models share the same space
     - Practical benefit: "pick up where you left off" even across topic boundaries
   - Links: ['https://docs.voyageai.com/']
   - tryIt: ['vai chat --memory-strategy hierarchical', 'vai explain memory-strategies']
