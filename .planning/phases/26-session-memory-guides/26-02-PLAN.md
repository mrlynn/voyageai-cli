Options:
  --db <name>               MongoDB database name
  --collection <name>       Collection with embedded documents
  --session <id>            Resume a previous chat session
  --llm-provider <name>     LLM provider: anthropic, openai, ollama
  --llm-model <name>        Specific LLM model to use
  --llm-api-key <key>       LLM API key
  --llm-base-url <url>      LLM API base URL (Ollama)
  --mode <mode>             Chat mode: pipeline (fixed RAG) or agent (tool-calling)
  --max-context-docs <n>    Max retrieved documents for context (default: 5)
  --max-turns <n>           Max conversation turns before truncation (default: 20)
  --no-history              Disable MongoDB persistence (in-memory only)
  --no-rerank               Skip reranking step
  --local                   Use local nano embeddings instead of Voyage API
  --embedding-model <name>  Embedding model
  --no-stream               Wait for complete response instead of streaming
  --system-prompt <text>    Override the system prompt
  --text-field <name>       Document text field name (default: "text")
  --filter <json>           MongoDB pre-filter for vector search
  --memory-strategy <name>  Memory strategy: sliding_window, summarization, hierarchical
  --estimate                Show estimated per-turn cost breakdown and exit
  --replay <id>             Replay a stored session for debugging
  --list                    List recent chat sessions and exit
  --all                     Include archived sessions in --list output
  --json                    Output JSON per turn (for scripting)
  -q, --quiet               Suppress decorative output

Slash commands: /help, /history, /memory, /clear, /quit, /exit, /stats

**docs/guides/memory-strategies.mdx** — Memory strategies guide (SESS-02):
- Frontmatter: title, description, sidebar_position
- Sections:
  - Overview: why memory management matters (LLM context windows are finite)
  - Token Budget: how the budget is allocated (system prompt, context docs, message, response reserve, remainder = history)
  - Sliding Window (default): most recent turns, fastest, no extra LLM calls. Best for short conversations.
  - Summarization: LLM-generated summaries of older turns, 60/40 recent/old split. Best for long conversations where early context matters.
  - Hierarchical: recent turns + summaries + cross-session recall. Best for multi-session workflows on related topics.
  - Decision Matrix: table comparing strategies on speed, cost, context preservation, cross-session support
  - Selecting a Strategy: `--memory-strategy <name>` flag and `/memory` slash command
  - Monitoring: `/memory` shows strategy, budget breakdown, utilization percentage
