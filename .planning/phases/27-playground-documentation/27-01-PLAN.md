    - Intro: The VAI Playground is a built-in browser UI that opens when you run `vai playground`. It provides tabs for experimenting with embedding, reranking, multimodal retrieval, chat, local inference, and more — all without leaving the terminal.

    ## Opening the Playground

    ```bash
    vai playground
    ```

    The playground opens in your default browser at `http://localhost:3000` (or the next available port). No additional setup is required beyond having `vai` installed.

    ## Tab Overview

    The playground is organized into four sidebar sections. Each section contains one or more tabs:

    ### Build

    | Tab | Purpose |
    |-----|---------|
    | Embed | Embed text and inspect the resulting vector |
    | Rerank | Score and reorder a list of candidate passages |
    | Multimodal | Embed images alongside text queries |
    | Generate | Generate code or scaffold snippets using LLM + embeddings |
    | **Chat** | RAG-powered conversational chat backed by a knowledge base |
    | Workflows | Build multi-step embedding pipelines |

    ### Evaluate

    | Tab | Purpose |
    |-----|---------|
    | Compare | Side-by-side semantic similarity comparison |
    | Benchmark | Latency and throughput benchmarks |
    | Optimize | Quantization and storage optimization analysis |
    | Explore | Guided feature walkthroughs |

    ### Reference

    | Tab | Purpose |
    |-----|---------|
    | Models | Voyage AI model catalog with pricing and specs |
    | About | Version information |

    ### Settings

    Configure API keys, database connection, and global preferences.

    ## Key Features

    **Zero API key for local inference** — The Local Inference tab runs `voyage-4-nano` on your machine via a Python bridge. No Voyage AI API key is required.

    **Persistent knowledge bases** — The Chat tab stores ingested documents in your MongoDB Atlas collection. Knowledge bases persist between playground sessions.

    **Light and dark mode** — Toggle between themes using the Settings tab or the theme toggle in the top-right corner.

    ## Welcome Banner

    Describe the onboarding banner (`#onboardingBanner`) that appears when first opening the Chat tab if no provider is configured:
    - Displays "Welcome to VAI Chat"
    - Auto-detects available services (Anthropic, OpenAI, Ollama, nano bridge)
    - Shows a recommended configuration based on detected services
    - Provides "Start Chatting" button to apply the recommended config, or "Configure Manually" to set options yourself

    ## Model Selector and Provider Badges

    Describe the Configuration section inside the KB panel:
    - **LLM Provider** — select from Anthropic, OpenAI, or Ollama. An Ollama health indicator dot shows connection status.
    - **Model** — select the specific LLM model for the chosen provider.
    - **Embedding model** — choose from `voyage-4-nano` (Free), `voyage-4-lite` ($0.02/1M tokens), `voyage-4` ($0.06/1M), or `voyage-4-large` ($0.12/1M).
    - **Provider badge** — a badge next to the embedding selector shows `LOCAL` (green) when `voyage-4-nano` is selected (runs on device, no API key needed) or `API` (blue) when a cloud model is selected.
    - **Nano bridge health dot** — shows whether the local inference bridge is running.

    ## Knowledge Base Panel

    Describe the collapsible KB panel on the left side of the chat tab:
    - **KB Info** — shows document count, chunk count, and storage size for the active knowledge base. Click the KB name to rename it.
    - **Files** — lists all ingested documents with their filenames.
    - **Upload** — three ingest methods:
      - *Files* — drag-and-drop or click to browse; accepts `.txt`, `.md`, `.pdf` up to 10 MB each
      - *Paste* — paste raw text with an optional title
      - *URL* — enter a URL to fetch and ingest a web page
    - **Instructions** — optional custom system prompt appended to every request.
    - **Configuration** — LLM provider, model, embedding model, API key, database, collection, max context documents, rerank toggle, chat mode, and memory strategy (see below).

    ## Memory Bar

    Describe `#chatMemoryBar`:
    - Horizontal progress bar showing "Memory: [bar] X%"
    - Displays what percentage of the token budget the current conversation history occupies
    - As the conversation grows, the bar fills left-to-right
    - When the bar approaches 100%, the active memory strategy takes effect (trimming or summarizing history)

    ## Turn State Indicator

    Describe `#chatStateIndicator`:
    - A colored dot + label below the messages area
    - Possible states: Ready (idle), Thinking (LLM processing), Streaming (response generating), Error
    - Matches the session turn states managed by the chat harness

    ## Memory Strategy Selector

    Describe `#chatMemoryStrategy` in the Configuration section:
    - **Sliding Window** (default) — keeps the most recent N turns, drops oldest when budget is full
    - **Summarization** — periodically summarizes older turns into a compressed block
    - **Hierarchical** — maintains both a rolling summary and a detailed recent window

    Cross-reference: See [Memory Strategies](../guides/memory-strategies.mdx) for detailed guidance on choosing a strategy.
