# Requirements: voyageai-cli v1.7 Bundled Knowledge Base

**Defined:** 2026-03-10
**Core Value:** A developer can go from `npm install` to a working vector search pipeline with zero API keys, zero accounts, and zero cost -- then seamlessly upgrade to the Voyage API when ready to scale.
**Spec:** `docs/vai-intent-driven-ux-spec.md` (sections 3-4)

## v1.7 Requirements

### KB Corpus

- [x] **CORP-01**: User can find ~30 pre-authored KB documents covering explainers, guides, reference, and examples
- [x] **CORP-02**: KB manifest.json indexes all documents with checksums, metadata, and versioning
- [x] **CORP-03**: KB documents are bundled in the npm package at `src/kb/corpus/` as offline fallback
- [x] **CORP-04**: Release workflow syncs KB snapshot from docs site and verifies manifest version match

### KB CLI

- [ ] **KBCL-01**: User can run `vai kb setup` to seed the bundled KB into MongoDB
- [ ] **KBCL-02**: User can run `vai kb status` to see collection stats, version, chunk count, and corpus source
- [ ] **KBCL-03**: User can run `vai kb update` to incrementally re-embed only changed documents
- [ ] **KBCL-04**: User can run `vai kb search <query>` to test-search the KB collection
- [ ] **KBCL-05**: User can run `vai kb reset` to drop and re-seed the KB (with confirmation)

### KB Seeding

- [ ] **SEED-01**: Seeding fetches manifest from docs site first, falls back to bundled snapshot on failure
- [ ] **SEED-02**: Seeding shows cost estimate and requires user confirmation before embedding
- [ ] **SEED-03**: Seeding chunks documents, embeds via Voyage API, inserts into Atlas with KB metadata schema
- [ ] **SEED-04**: Seeding creates vector search index on the `vai_kb` collection
- [ ] **SEED-05**: Seeding tracks corpus source (remote vs bundled) and version in config

### Self-Demonstrating Chat

- [ ] **CHAT-01**: `vai chat` with no collection configured falls back to `vai_kb` with informative banner
- [ ] **CHAT-02**: KB-mode chat uses a tuned system prompt that positions vai as a guide
- [ ] **CHAT-03**: Chat displays starter question chips on first open (rotated per session)
- [ ] **CHAT-04**: Chat shows collapsible "How this answer was found" panel per assistant message
- [ ] **CHAT-05**: User can transition from KB chat to their own collection via inline config panel

## Future Requirements

### Persona Modes (v1.8)

- **MODE-01**: User can select Explore/Build/Expert mode at first run
- **MODE-02**: Mode persists in config and affects chat, tab order, and guidance density
- **MODE-03**: User can switch mode from sidebar footer or Settings
- **MODE-04**: First-run overlay appears when no mode is set

### Adaptive Home (v1.9)

- **HOME-01**: Home page renders mode-specific layouts (Explore/Build/Expert)
- **HOME-02**: Three flagship demo cards with one-click launch (Explore mode)
- **HOME-03**: Setup status widget and guided path cards (Build mode)
- **HOME-04**: Compact Quick Actions + Recent (Expert mode)
- **HOME-05**: Onboarding checklist for Explore mode

## Out of Scope

| Feature | Reason |
|---------|--------|
| Persona modes (Explore/Build/Expert) | Deferred to v1.8 -- requires KB foundation first |
| First-run overlay | Deferred to v1.8 -- depends on mode system |
| Adaptive home page | Deferred to v1.9 -- depends on mode system |
| Flagship demo cards | Deferred to v1.9 -- depends on adaptive home |
| Onboarding checklist | Deferred to v1.9 -- depends on mode system |
| Mode-aware system prompt injection ({{ mode }}) | Deferred to v1.8 -- KB-mode prompt works standalone |
| In-browser model inference (WASM/WebGPU) | Model too large; Python bridge is validated approach |
| Multi-KB simultaneous search | Complex routing; single active KB sufficient |
