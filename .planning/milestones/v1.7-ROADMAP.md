## Overview

Ship vai's own documentation as a pre-embedded knowledge base that enables zero-setup chat from first launch. The KB corpus is authored in Docusaurus, bundled in the npm package, seeded into Atlas via the existing embed/ingest pipeline, and surfaced through a self-demonstrating chat experience with starter chips, source attribution, and transition to user data.

## Phase Details

### Phase 30: KB Corpus & Manifest
**Goal**: Developers can browse vai's knowledge base as documentation and the corpus ships inside the npm package ready for seeding
**Depends on**: Nothing (first phase of v1.7)
**Requirements**: CORP-01, CORP-02, CORP-03, CORP-04
**Success Criteria** (what must be TRUE):
  1. ~30 Markdown documents exist covering explainers, guides, reference, and examples categories
  2. manifest.json lists every document with title, category, path, checksum, and corpus version
  3. Documents are present at `src/kb/corpus/` in the npm package and accessible at runtime
  4. Release workflow syncs KB snapshot from docs site and verifies manifest version matches package version

### Phase 31: KB Seeding Pipeline
**Goal**: Users can seed the bundled KB into MongoDB Atlas with full embedding and indexing, using the existing embed/ingest infrastructure
**Depends on**: Phase 30
**Requirements**: SEED-01, SEED-02, SEED-03, SEED-04, SEED-05
**Success Criteria** (what must be TRUE):
  1. Seeding fetches manifest from docs.vaicli.com first and falls back to bundled snapshot when offline
  2. User sees a cost estimate and must confirm before any Voyage API calls are made
  3. Documents are chunked, embedded via Voyage API, and inserted into the `vai_kb` collection with KB metadata schema
  4. A vector search index is created on the `vai_kb` collection automatically
  5. Config records corpus source (remote vs bundled) and version after seeding completes

### Phase 32: KB CLI Commands
**Goal**: Users can manage the KB lifecycle entirely from the CLI with setup, status, update, search, and reset commands
**Depends on**: Phase 31
**Requirements**: KBCL-01, KBCL-02, KBCL-03, KBCL-04, KBCL-05
**Success Criteria** (what must be TRUE):
  1. `vai kb setup` seeds the KB into Atlas (invokes seeding pipeline from Phase 31)
  2. `vai kb status` displays collection stats including version, chunk count, corpus source, and index health
  3. `vai kb update` re-embeds only documents whose checksums have changed since last seed
  4. `vai kb search <query>` returns ranked KB results with scores and source document references
  5. `vai kb reset` drops and re-seeds the KB collection after user confirmation

### Phase 33: Self-Demonstrating Chat
**Goal**: Users experience vai as a self-demonstrating tool from first launch, with zero-setup KB chat, guided discovery, and a clear path to their own data
**Depends on**: Phase 32
**Requirements**: CHAT-01, CHAT-02, CHAT-03, CHAT-04, CHAT-05
**Success Criteria** (what must be TRUE):
  1. `vai chat` with no collection configured automatically uses `vai_kb` and shows an informative banner explaining KB mode
  2. KB-mode chat uses a tuned system prompt that positions vai as a knowledgeable guide about its own capabilities
  3. Starter question chips appear on first chat open, rotated per session, and are clickable to submit
  4. Each assistant message includes a collapsible "How this answer was found" panel showing source chunks and scores
  5. User can transition from KB chat to their own collection via an inline config panel without leaving chat
