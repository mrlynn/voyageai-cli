Create `src/demos/nano.js` with these exports and functions:

**Module structure:**
- `module.exports = { runNanoDemo }` -- single async function receiving `opts` ({ verbose, pause })
- Copy `theory()` and `step()` helpers from demo.js at the top (they are not exported from demo.js, so duplicate them -- they are 10 lines total)

**Sample texts (9 texts, 3 clusters):**
```
Database cluster:
  1. "Create an index on the email field for faster lookups"
  2. "Shard the collection across regions for horizontal scaling"
  3. "Use aggregation pipelines to transform and filter documents"

Auth cluster:
  4. "JWT tokens expire after 24 hours and must be refreshed"
  5. "Hash passwords with bcrypt before storing in the database"
  6. "OAuth2 authorization code flow redirects to the callback URL"

Caching cluster:
  7. "Redis stores key-value pairs in memory for sub-millisecond reads"
  8. "Set a TTL on cache entries to prevent stale data"
  9. "Invalidate the cache when the underlying data changes"
```

Short labels for matrix headers (2-3 words each):
`["db-index", "db-shard", "db-aggreg", "auth-jwt", "auth-hash", "auth-oauth", "cache-redis", "cache-ttl", "cache-inval"]`

Cluster assignment array: `[0, 0, 0, 1, 1, 1, 2, 2, 2]` (for dimension comparison stats).

**checkNanoPrerequisites():**
- Call `checkVenv()`, `checkDeps()`, `checkModel()` from `nano-health.js`
- Return `{ ok: boolean, checks: { venv, deps, model } }`
- If not ok, print each failing check with its hint, then print a summary: "Run `vai nano setup` to get started." and return (do NOT process.exit -- let the caller handle it)

**cosineSimilarity(a, b):**
- Standard dot product / (magnitude_a * magnitude_b)
- 5 lines, no dependencies

**displaySimilarityMatrix(embeddings, labels):**
- Compute full NxN pairwise cosine similarity
- Print header row with labels padded to align
- Print each row with color coding: `pc.green` for >= 0.7 (high), `pc.yellow` for >= 0.4 (medium), `pc.dim` for < 0.4 (low), `pc.dim` for diagonal (always 1.00)
- Truncate scores to 2 decimal places, padStart(6) for alignment
- Print summary line: "Within-cluster: 0.XX-0.XX | Cross-cluster: 0.XX-0.XX"
- Compute within/cross ranges from cluster assignment

**compareDimensions(texts, clusterAssignment, dimensions):**
- Takes array of dimensions [256, 1024, 2048]
- For each dimension, call `generateLocalEmbeddings(texts, { dimensions: dim })`
- Compute avg within-cluster and avg cross-cluster similarity
- Compute memory estimate: `dim * 4 * 1000 / 1024` KB per 1K vectors (float32)
- Return array of `{ dim, withinAvg, crossAvg, memoryKB }`

**displayDimensionTable(results):**
- Table with columns: Dimensions | Within-Cluster | Cross-Cluster | Separation | Memory/1K
- "Separation" = withinAvg - crossAvg (how distinguishable clusters are)
- Print summary line interpreting the tradeoff, e.g., "256 dims retains ~XX% separation at 1/8 memory"

**runNanoDemo(opts) flow:**
1. Check prerequisites. If not ready, print guidance and return (not exit).
2. Print header: `pc.bold('  Local Embeddings Demo')` + rule line
3. Print intro text explaining zero-dependency local inference
4. `theory(verbose, ...)` about how nano embeddings work locally
5. **Step 1: Similarity Matrix**
   - Print step header: `pc.bold('  Step 1: Embedding sample texts...')`
   - Wrap first `generateLocalEmbeddings()` call in `ui.spinner('Loading voyage-4-nano model and embedding 9 texts...')` -- succeed with 'Model loaded — embeddings generated'
   - Display the 9x9 matrix with `displaySimilarityMatrix()`
   - `theory(verbose, ...)` explaining cosine similarity and clustering
6. **Step 2: Dimension Comparison**
   - Print step header: `pc.bold('  Step 2: Comparing MRL dimensions...')`
   - Print brief "Comparing dimensions..." status (no spinner -- model already loaded, calls are fast)
   - Call `compareDimensions()` with [256, 1024, 2048]
   - Display table with `displayDimensionTable()`
   - `theory(verbose, ...)` explaining MRL (Matryoshka Representation Learning)
7. Return the cached embeddings and labels so the caller (Plan 02) can use them for the REPL without re-embedding:
   ```javascript
   return { embeddings, labels, sampleTexts };
   ```

**Important implementation details:**
- Use `require('picocolors')` for colors (project standard, NOT chalk)
- Use lazy requires: `const { generateLocalEmbeddings } = require('../nano/nano-local')` inside the function
- The `ui` import: `const ui = require('../lib/ui')`
- Do NOT call `getBridgeManager().shutdown()` in this module -- that happens in demo.js after the full flow completes
- Do NOT import readline or handle REPL -- that is Plan 02
- Matrix must fit in 80-column terminal. 9 labels at max 10 chars + 9 scores at 6 chars each = ~64 + padding. Test with the chosen labels.
