<objective>
Extend the robot animation API to support elapsed time counters alongside pose labels, and add a `startWaving` moment for chat startup.

Purpose: The chat command currently uses `createTimedSpinner` which shows elapsed seconds (e.g., "Searching knowledge base... 2.3s"). Robot animations must replicate this elapsed timer feature so the replacement is a UX upgrade, not a downgrade. The existing `animateRobot` renders a static label but has no timer. This plan adds timer support to the animation engine and exposes it through the moments API.

Output: Extended `animateRobot` with `showElapsed` option, new `startWaving` moment, unit tests.
</objective>

  <name>Task 1: Add elapsed timer support to animateRobot</name>
  <files>src/lib/robot.js</files>
  <action>
Extend the `animateRobot` function to accept a `showElapsed: boolean` option (default `false`). When `showElapsed` is true:

1. Record `startTime = Date.now()` when animation begins
2. On each frame redraw, compute elapsed seconds: `((Date.now() - startTime) / 1000).toFixed(1)`
3. Append the elapsed time after the label in dim styling: `label + '  ' + dim(elapsed + 's')` where `dim` uses picocolors `pc.dim()` (already available via the colorize helper pattern, but use raw ANSI dim `\x1b[2m...\x1b[0m` to stay consistent with the file's no-picocolors approach)
4. The elapsed text should render on the same midLine as the label, after the label text

Updated signature:
```javascript
function animateRobot(poseName, { indent = 2, label = '', showElapsed = false } = {})
```

In the `draw()` function, change the label rendering from:
```javascript
if (label) {
  lines[midLine] += `  ${colorize(COLORS.teal, label)}`;
}
```
to:
```javascript
if (label || showElapsed) {
  let labelText = label ? `  ${colorize(COLORS.teal, label)}` : '';
  if (showElapsed) {
    const elapsed = ((Date.now() - startTime) / 1000).toFixed(1);
    labelText += `  \x1b[2m${elapsed}s\x1b[0m`;
  }
  lines[midLine] += labelText;
}
```

Do NOT change the return type or `stop()` behavior -- just the visual output during animation frames.
