  <files>src/lib/chat-ui.js</files>
  <action>
Add three new exported functions to chat-ui.js for turn visual styling. These use picocolors (already imported) for consistency with the rest of the file.

1. **renderUserPrompt()** (TURN-01):
   Returns a styled prompt string for the readline prompt. Use a teal-colored arrow or chevron to distinguish from assistant output:
   ```javascript
   function renderUserPrompt() {
     return pc.bold(pc.green('\u276F '));  // ❯ (heavy right-pointing angle)
   }
   ```
   This replaces the current `pc.green('> ')`. The bold green chevron is visually distinct from assistant text (which is plain white/default). Keep it compact -- the prompt should be 2 visible chars ("chevron space").

2. **renderAssistantLabel()** (TURN-02):
   Returns a styled label string to print before the assistant's streaming response begins:
   ```javascript
   function renderAssistantLabel() {
     return pc.dim('\u2500\u2500 ') + pc.bold(pc.cyan('vai')) + pc.dim(' \u2500\u2500');
   }
   ```
   This produces something like `── vai ──` in dim/cyan styling. It's a compact label that signals "assistant is responding now" without being verbose. Print this on its own line before the first streaming chunk.

3. **renderTurnDivider()** (TURN-03):
   Returns a visual divider string for between turns:
   ```javascript
   function renderTurnDivider() {
     return pc.dim('\u2500'.repeat(40));
   }
   ```
   A dim horizontal line (40 chars) that provides clear turn separation without being heavy. This prints after the assistant's response (and sources, if any) completes, before the next user prompt.

4. Export all three functions from the module.exports at the bottom of the file.

Important: These helpers return strings. The caller (chat.js) decides when to console.log them and handles the interactive/non-interactive guard.
