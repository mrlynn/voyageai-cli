**Change 1 - Restore active class in updateEmbedBadge() (around line 16258-16263):**

**Change 2 - Add global toggleRerank() function and inline onclick as race-condition fallback:**

(a) Add a global function near the other chat settings functions (after saveChatSettings around line 16350):

```javascript
function toggleRerank() {
  const btn = document.getElementById('chatRerank');
  if (btn && !btn.disabled) {
    btn.classList.toggle('active');
    saveChatSettings();
  }
}
```

(b) On the button HTML at line 8883, add an onclick attribute:

```html
<button class="settings-toggle active" id="chatRerank" type="button" onclick="toggleRerank()"></button>
```

This ensures the rerank button works even if the addEventListener in initSettings() has not yet fired due to the init race condition. The addEventListener handler will also work fine alongside the onclick -- clicking will fire both, but since classList.toggle is idempotent per call, having the addEventListener handler is harmless (it will double-toggle, so we should REMOVE the addEventListener block in initSettings() to avoid double-toggling).

(c) REMOVE the addEventListener block in initSettings() (lines 14226-14232) to prevent double-toggle. The onclick handler is the sole click handler now.
