Add two new route handlers inside `handleRAGRequest()`, placed BEFORE the "Not a RAG endpoint" return at the bottom:

**POST /api/rag/ingest-text** -- accepts JSON `{ text, kbName, title? }`:
1. Validate: text is non-empty string, kbName exists in KB metadata collection
2. Use the same chunking logic as the existing file ingest (split by `\n\n+`, accumulate up to 1000 words per chunk)
3. Extract this chunking logic into a reusable `chunkText(content)` function at module level (refactor the existing `/api/rag/ingest` endpoint to also use it)
4. Stream NDJSON progress: emit `{type:'progress', stage:'chunking', current, total}` then `{type:'progress', stage:'embedding', current, total}` then `{type:'complete', ...}`
5. For each chunk, call `generateEmbeddings(chunk, 'voyage-4-large')` (same as existing ingest), insert into `kb_${kbName}_docs` with `fileName` set to title or `"pasted-text-{timestamp}"`
6. Update KB metadata counts via $inc

**POST /api/rag/ingest-url** -- accepts JSON `{ url, kbName }`:
1. Validate: url starts with http:// or https://, kbName exists
2. Fetch the URL using Node's built-in `fetch()` (available in Node 18+). Set a 15-second timeout using AbortController.
3. Extract text content: if response Content-Type contains 'text/html', strip HTML tags with a simple regex (`content.replace(/<script[\s\S]*?<\/script>/gi, '').replace(/<style[\s\S]*?<\/style>/gi, '').replace(/<[^>]+>/g, ' ').replace(/\s+/g, ' ').trim()`). If plain text, use as-is.
4. Use `chunkText()` to chunk, then embed and store same as ingest-text
5. Set `fileName` to the URL hostname + path (truncated to 80 chars)
6. Stream NDJSON progress with stage indicators

Both endpoints: follow the existing NDJSON streaming pattern, update KB metadata via $inc after completion.
