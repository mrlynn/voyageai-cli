**src/lib/ingestion/index.ts** — Orchestration:
```typescript
export interface IngestionResult {
  sourceId: string;
  chunkCount: number;
  documentCount: number;
  fingerprint: string;
  durationMs: number;
  previousChunkCount: number; // 0 if first index
}

export async function ingestSource(sourceId: string): Promise<IngestionResult>
// 1. Load source from MongoDB (getKnowledgeCollection)
// 2. Set source status to 'indexing', clear errorMessage
// 3. Based on source.type:
//    - 'file': readFile(source.sourcePath), chunkText with appropriate strategy, fingerprintFile()
//    - 'url': crawlUrl(source.sourcePath), chunkText with 'paragraph' strategy
//    - 'codebase': readCodebaseFiles(source.sourcePath), chunk each file with 'fixed' strategy (size 400)
//    - 'text': source.sourcePath contains the text content directly, chunk with 'paragraph'
// 4. Embed all chunks in batches (generateEmbeddings)
// 5. Delete existing chunks for this sourceId (getChunksCollection)
// 6. Insert new IndexedChunk documents (with id=crypto.randomUUID(), sourceId, content, embedding, etc.)
// 7. Create SourceVersion snapshot (getVersionsCollection) with diffSummary vs previous chunk count
// 8. Update source: status='indexed', chunkCount, documentCount, fingerprint, lastIndexedAt, updatedAt
// 9. Return IngestionResult
// On any error: set source status='error', errorMessage=err.message, re-throw

export async function checkAndReindex(sourceId: string): Promise<{ reindexed: boolean; reason: string; result?: IngestionResult }>
// 1. Load source. If status !== 'indexed', proceed with full index.
// 2. Compute current fingerprint (fingerprintFile or fingerprintUrl based on type)
// 3. Compare to source.fingerprint
// 4. If same: return { reindexed: false, reason: 'fingerprint unchanged' }
// 5. If different: call ingestSource(sourceId), return { reindexed: true, reason: 'content changed', result }
```

**Chunk strategy selection guide** (use in ingestSource):
- .md files → 'markdown' strategy
- .ts/.tsx/.js/.jsx files → 'fixed' strategy (size=400, overlap=40)
- .json/.yaml files → 'fixed' strategy (size=600, overlap=0)
- General text / URL content → 'paragraph' strategy
- Unknown → 'paragraph' strategy
