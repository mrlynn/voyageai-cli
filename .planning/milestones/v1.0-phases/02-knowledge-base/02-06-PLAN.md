Create src/nano/nano-health.js with individual health checks and three command handlers.

**CHECKS array** (following doctor.js pattern):
```javascript
const CHECKS = [
  { key: 'python', name: 'Python 3.10+', fn: checkPython },
  { key: 'venv', name: 'Virtual Environment', fn: checkVenv },
  { key: 'deps', name: 'Python Dependencies', fn: checkDeps },
  { key: 'model', name: 'voyage-4-nano Model', fn: checkModel },
  { key: 'device', name: 'Compute Device', fn: checkDevice },
];
```

**checkPython():**
- Try `python3 --version` via execFileSync (encoding: 'utf8', timeout: 5000, stdio: 'pipe')
- Parse version with `/Python\s+(\d+)\.(\d+)/`
- If major < 3 || minor < 10: return `{ ok: false, message: '<version> (requires 3.10+)', hint: 'Install Python 3.10+: https://www.python.org/downloads/' }`
- On success: return `{ ok: true, message: 'Python <version>', hint: null }`
- On error (ENOENT): return `{ ok: false, message: 'Not found', hint: 'Install Python 3.10+: https://www.python.org/downloads/' }`

**checkVenv():**
- Check `fs.existsSync(VENV_PYTHON)` where VENV_PYTHON is from nano-setup.js exports (or define same constant locally)
- ok: `{ ok: true, message: VENV_DIR, hint: null }`
- not ok: `{ ok: false, message: 'Not found', hint: 'Run: vai nano setup' }`

**checkDeps():**
- If venv doesn't exist: return `{ ok: false, message: 'Venv not found', hint: 'Run: vai nano setup' }`
- Run `execFileSync(VENV_PYTHON, ['-c', 'import sentence_transformers; import torch; print(sentence_transformers.__version__, torch.__version__)'], { encoding: 'utf8', timeout: 10000, stdio: 'pipe' })`
- Parse output for versions
- ok: `{ ok: true, message: 'sentence-transformers <ver>, torch <ver>', hint: null }`
- fail: `{ ok: false, message: 'Missing or broken', hint: 'Run: vai nano setup --force' }`

**checkModel():**
- Check MODEL_CACHE_DIR exists and has subdirectory containing 'voyage'
- ok: `{ ok: true, message: MODEL_CACHE_DIR, hint: null }`
- not ok: `{ ok: false, message: 'Not downloaded', hint: 'Run: vai nano setup' }`

**checkDevice():**
- If venv doesn't exist: return `{ ok: false, message: 'Cannot detect (no venv)', hint: 'Run: vai nano setup' }`
- Run `execFileSync(VENV_PYTHON, ['-c', "import torch; print('cuda' if torch.cuda.is_available() else 'mps' if torch.backends.mps.is_available() else 'cpu')"], { encoding: 'utf8', timeout: 10000, stdio: 'pipe' })`
- Return `{ ok: true, message: '<device> (e.g., cpu, cuda, mps)', hint: null }`
- Always ok=true if device detection succeeds (cpu is fine, just informational)

**runStatus(options = {}):**
- Run all CHECKS sequentially
- For each: display `checkMark(result.ok) name: message` using picocolors
- If result.hint: display `  hint: <hint>` in yellow
- If `options.json`: output `JSON.stringify(results)` and return (no colored output)
- At end: summary line "X/5 checks passed"

**runTest():**
- Use getBridgeManager() from nano-manager.js
- Log "Running smoke test..."
- `const start = performance.now()`
- Call `manager.embed('The quick brown fox jumps over the lazy dog', { inputType: 'document', dimensions: 1024 })`
- `const elapsed = (performance.now() - start).toFixed(0)`
- Display: "Embedding generated in {elapsed}ms"
- Display: "Dimensions: {result.dimensions}" (use ui.label if available, else console.log)
- Display: "Vector length: {result.embeddings[0].length}"
- Display: "First 5 values: {result.embeddings[0].slice(0, 5).map(v => v.toFixed(6)).join(', ')}"
- Call `manager.shutdown()` in finally block
- On error: display formatted error and exit(1)

**runInfo():**
- Display model name: "voyageai/voyage-4-nano"
- Display model cache path: MODEL_CACHE_DIR
- If model exists: calculate and display directory size (reuse getDirSize pattern from nano-setup.js or implement inline)
- If venv exists: detect device (call checkDevice) and display
- Display venv path: VENV_DIR
- Display requirements.txt path: path.join(__dirname, 'requirements.txt')
- If nothing is set up: display "Not set up. Run: vai nano setup"

**Exports:** `{ runStatus, runTest, runInfo, checkPython, checkVenv, checkDeps, checkModel, checkDevice }`

Export individual check functions for testability (TEST-03 will mock these).
