  <name>Task 1: Create Node.js bridge manager with singleton lifecycle</name>
  <files>src/nano/nano-manager.js</files>
  <action>
    Create `src/nano/nano-manager.js` -- the singleton manager for the Python bridge subprocess.

    Class: `NanoBridgeManager`

    Private state:
    - `#process` -- the child_process instance (null when not running)
    - `#pending` -- Map of request id -> {resolve, reject, timer} for in-flight requests
    - `#idleTimer` -- setTimeout handle for idle shutdown
    - `#ready` -- Promise that resolves when Python sends the ready signal
    - `#readyResolve` / `#readyReject` -- stored resolve/reject for #ready
    - `#buffer` -- string buffer for partial stdout lines
    - `#crashCount` -- number of consecutive crashes (reset on successful response)
    - `#bridgeVersion` -- version reported by Python in ready signal
    - `#cleanupRegistered` -- boolean, prevent double-registering exit handlers

    Constructor:
    - Initialize all state to defaults
    - IDLE_TIMEOUT = 30_000 (30 seconds)
    - REQUEST_TIMEOUT = 60_000 (60 seconds, accommodates cold model load)

    Public methods:

    `async embed(texts, options = {})`:
    - Call `#ensureProcess()` to spawn if needed, await ready
    - Create request via nano-protocol.js createRequest('embed', { texts, input_type: options.inputType || 'document', truncate_dim: options.dimensions || 2048, precision: options.precision || 'float32' })
    - serializeRequest and write to process.stdin
    - If stdin.write fails, reject with NANO_STDIN_WRITE_FAILED
    - Return a Promise stored in #pending map with per-request timeout (REQUEST_TIMEOUT)
    - Reset idle timer on each call

    `async shutdown()`:
    - Clear idle timer
    - If process exists: send SIGTERM, set a 5-second fallback SIGKILL timer
    - Reject all pending requests with NANO_PROCESS_CRASH
    - Set #process = null

    `isRunning()`:
    - Returns boolean: #process !== null and not killed

    Private methods:

    `async #ensureProcess()`:
    - If #process exists and running, return
    - Determine Python path: check if ~/.vai/nano-env/bin/python3 exists (fs.existsSync), fall back to 'python3'
    - If neither found, throw createNanoError('NANO_PYTHON_NOT_FOUND')
    - Spawn with: spawn(pythonPath, [bridgeScriptPath], { stdio: ['pipe', 'pipe', 'pipe'], env: { ...process.env, PYTHONUNBUFFERED: '1', PYTHONDONTWRITEBYTECODE: '1' }, cwd: nanoDir })
    - Set up #ready promise
    - Attach stdout 'data' handler -> #handleData(chunk)
    - Attach stderr 'data' handler -> collect stderr for error reporting
    - Attach 'error' handler -> reject ready, reject all pending with NANO_SPAWN_FAILED
    - Attach 'close' handler -> #handleClose(code, signal)
    - Register cleanup handlers (once only, check #cleanupRegistered)
    - Await #ready (the ready signal from Python)
    - Check bridge version from ready signal against package.json version; throw NANO_BRIDGE_VERSION_MISMATCH if mismatch

    `#handleData(chunk)`:
    - Append chunk.toString() to #buffer
    - Split on '\n', process complete lines
    - For each complete line: parseLine() from nano-protocol.js
    - If type === 'ready': store bridge version, device; resolve #ready
    - If type === 'result': find pending by id, clearTimeout its timer, resolve with the response
    - If type === 'error': find pending by id, clearTimeout its timer, reject with createNanoError based on response.code
    - Reset #crashCount to 0 on successful result

    `#handleClose(code, signal)`:
    - If code !== 0 and #pending has entries:
      - Increment #crashCount
      - If #crashCount < 2: attempt auto-restart (#ensureProcess + re-send pending requests)
      - If #crashCount >= 2: reject all pending with NANO_PROCESS_CRASH
    - Set #process = null

    `#resetIdleTimer()`:
    - clearTimeout(#idleTimer)
    - #idleTimer = setTimeout(() => this.shutdown(), IDLE_TIMEOUT)

    `#registerCleanup()`:
    - If #cleanupRegistered, return
    - const cleanup = () => { if (this.#process) { this.#process.kill('SIGKILL'); this.#process = null; } }
    - process.on('exit', cleanup)
    - process.on('SIGINT', () => { cleanup(); process.exit(130); })
    - process.on('SIGTERM', () => { cleanup(); process.exit(143); })
    - process.on('uncaughtException', (e) => { cleanup(); throw e; })
    - Set #cleanupRegistered = true

    Module-level singleton:

    ```javascript
    let _instance = null;
    function getBridgeManager() {
      if (!_instance) {
        _instance = new NanoBridgeManager();
      }
      return _instance;
    }
    ```

    Export: { NanoBridgeManager, getBridgeManager }

    Version check: Read package.json version at module load time using require('../../package.json').version. Compare against bridge_version from ready signal.

    Use CommonJS. Use node:child_process, node:path, node:fs, node:os, node:crypto (via nano-protocol.js).

    Create `scripts/sync-nano-version.js` -- a Node.js script that syncs BRIDGE_VERSION in nano-bridge.py with the version from package.json.

    The script:
    1. Read package.json -> extract version string
    2. Read src/nano/nano-bridge.py -> find the line `BRIDGE_VERSION = "..."` using regex
    3. Replace the version string with the one from package.json
    4. Write the updated nano-bridge.py back
    5. Print: `Synced BRIDGE_VERSION to {version}` on success
    6. Exit 1 with error message if nano-bridge.py not found or BRIDGE_VERSION line not found

    Regex pattern: /^BRIDGE_VERSION\s*=\s*"[^"]*"/m
    Replacement: `BRIDGE_VERSION = "${version}"`

    This script is intended to be called by `npm version` hook (added in Phase 2). For now, it just needs to exist and work when run manually:
    ```bash
    node scripts/sync-nano-version.js
    ```

    ```bash
    node scripts/sync-nano-version.js --check
    ```
