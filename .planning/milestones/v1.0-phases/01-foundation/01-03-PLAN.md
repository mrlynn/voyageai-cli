  <files>src/nano/nano-errors.js, src/nano/nano-protocol.js</files>
  <action>
    Create `src/nano/` directory.

    Create `src/nano/nano-errors.js` with the full error taxonomy. Each error has:
    - `code`: string constant (e.g., 'NANO_PYTHON_NOT_FOUND')
    - `message`: human-readable description (string or function for dynamic messages)
    - `fix`: copy-pasteable remediation command

    Error codes to include (~10 distinct):
    1. NANO_PYTHON_NOT_FOUND -- Python 3.10+ not found on PATH
       fix: 'Install Python 3.10+: https://www.python.org/downloads/'
    2. NANO_PYTHON_VERSION -- Python found but version < 3.10
       fix: 'Upgrade to Python 3.10+: https://www.python.org/downloads/'
    3. NANO_VENV_MISSING -- venv at ~/.vai/nano-env/ does not exist
       fix: 'Run: vai nano setup'
    4. NANO_DEPS_MISSING -- Python dependencies not installed in venv
       fix: 'Run: vai nano setup'
    5. NANO_MODEL_NOT_FOUND -- voyage-4-nano model not cached locally
       fix: 'Run: vai nano setup'
    6. NANO_BRIDGE_VERSION_MISMATCH -- BRIDGE_VERSION in Python != package.json version
       fix: 'Run: vai nano setup --force'
       message is a function: (expected, actual) => `Bridge version mismatch: expected ${expected}, got ${actual}`
    7. NANO_PROCESS_CRASH -- Python subprocess crashed unexpectedly
       fix: 'Run: vai nano setup --force'
    8. NANO_JSON_PARSE_ERROR -- Received malformed JSON from bridge
       fix: 'Run: vai nano setup --force'
    9. NANO_TIMEOUT -- Bridge did not respond within timeout
       fix: 'Run: vai nano status to check health'
    10. NANO_SPAWN_FAILED -- Failed to start Python bridge process
        fix: 'Run: vai nano status to diagnose'
    11. NANO_STDIN_WRITE_FAILED -- Failed to write to bridge stdin
        fix: 'Restart the CLI and try again'

    Export:
    - `NANO_ERRORS` object (code -> {message, fix})
    - `createNanoError(code, ...args)` -- returns Error with code, message, fix properties
    - `formatNanoError(error)` -- formats using ui.error() and returns the error string with remediation

    Use CommonJS (require/module.exports) to match project convention.

    Create `src/nano/nano-protocol.js` with NDJSON helpers:

    - `createRequest(type, payload)` -- builds a request envelope with: id (crypto.randomUUID()), type, and spread payload fields. Returns the object (not serialized).
    - `serializeRequest(requestObj)` -- JSON.stringify(requestObj) + '\n'
    - `parseLine(line)` -- JSON.parse a single line, returns parsed object. Throws with NANO_JSON_PARSE_ERROR on failure.
    - `validateResponse(response, requestId)` -- checks response has correct id and type field. Returns true or throws.
    - `ENVELOPE_TYPES` -- constant object: { EMBED: 'embed', RESULT: 'result', ERROR: 'error', READY: 'ready' }

    Use `node:crypto` for randomUUID(). CommonJS style.
  </action>

  <name>Task 2: Create Python bridge script and requirements.txt</name>
  <files>src/nano/nano-bridge.py, src/nano/requirements.txt</files>
  <action>
    Create `src/nano/requirements.txt` with compatible-release pins (per project decision):

    Create `src/nano/nano-bridge.py` -- the complete Python bridge subprocess.

    Structure (single file per user decision):

    1. Constants at top:
       - BRIDGE_VERSION = "1.31.0" (matches package.json version)
       - MODEL_NAME = "voyageai/voyage-4-nano"

    2. `detect_device()` function:
       - Import torch
       - Check torch.cuda.is_available() -> "cuda"
       - Check hasattr(torch.backends, "mps") and torch.backends.mps.is_available() -> "mps"
       - Fallback -> "cpu"

    3. `load_model(device)` function:
       - Import SentenceTransformer from sentence_transformers
       - Load with trust_remote_code=True and specified device
       - If model not found locally (OSError or similar), return error dict with code MODEL_NOT_FOUND
       - Return the model object on success

    4. `handle_embed(model, request)` function:
       - Extract texts, input_type (default "document"), truncate_dim (default 2048), precision (default "float32")
       - Select encode_query or encode_document based on input_type
       - Handle single text string -> wrap in list
       - Call encode function with truncate_dim and precision params
       - Return response envelope: {id, type: "result", embeddings: embeddings.tolist(), dimensions: truncate_dim, usage: {total_tokens: estimated}}
       - Token count estimation: sum of len(text.split()) for each text (rough approximation)

    5. `main()` function:
       - Use `while True: line = sys.stdin.readline()` (NOT `for line in sys.stdin` -- avoids hidden buffering per research pitfall 3)
       - Send ready signal immediately: {type: "ready", bridge_version: BRIDGE_VERSION, device: detected_device, model: MODEL_NAME}
       - Flush stdout after every write: sys.stdout.write(json.dumps(msg) + "\n"); sys.stdout.flush()
       - Lazy-load model on first embed request (not at startup -- per research pitfall 5)
       - If model loading fails, return {id, type: "error", code: "MODEL_NOT_FOUND", message: "..."}
       - If JSON parse fails, return {type: "error", code: "JSON_PARSE_ERROR", message: "..."}
       - If any other exception, return {id (if available), type: "error", code: "BRIDGE_ERROR", message: str(e)}
       - Break loop when readline() returns empty string (stdin closed = parent exited)

    6. Add shebang line: #!/usr/bin/env python3
    7. Add module docstring: """nano-bridge.py - voyage-4-nano NDJSON bridge for voyageai-cli"""

    Critical buffering mitigations (per research):
    - Do NOT set sys.stdout to line-buffered in Python code (rely on PYTHONUNBUFFERED=1 from Node side + explicit flush)
    - Call sys.stdout.flush() after EVERY sys.stdout.write()
    - Use sys.stdin.readline() not `for line in sys.stdin`
  </action>
