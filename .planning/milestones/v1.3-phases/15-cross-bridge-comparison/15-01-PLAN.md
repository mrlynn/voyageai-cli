<task type="auto">
  <name>Task 1: Add POST /api/nano/crossbridge endpoint</name>
  <files>src/lib/playground-nano-api.js</files>
  <action>
Add a new route handler inside handleNanoRequest for POST /api/nano/crossbridge. This endpoint:

1. Accepts JSON body: { text: string, dimension?: number (default 1024) }
2. Validates text (required, string, max 10000 chars) and dimension (must be in VALID_DIMENSIONS)
3. Checks nano readiness (checkVenv + checkModel), returns 503 with NANO_NOT_READY if not ready
4. Generates nano embedding: await generateLocalEmbeddings([body.text], { dimensions: dimension })
5. Calls the Voyage API for the same text using context.generateCloudEmbeddings (a new context dependency). This function will be the existing generateEmbeddings from api.js, passed in by playground.js. If API call fails (no key, auth error), returns the response with nanoEmbedding only and apiError field.
6. If both succeed, computes cosine similarity between the two vectors using cosineSimilarity (already imported)
7. Returns JSON:
```json
{
  "text": "...",
  "dimension": 1024,
  "nano": { "embedding": [...], "model": "voyage-4-nano", "latency_ms": N },
  "api": { "embedding": [...], "model": "voyage-3-large", "latency_ms": N },
  "similarity": 0.9987,
  "latency_ms": N
}
```
If API fails: `"api": null, "apiError": "error message", "similarity": null`

Place this route BEFORE the final `return false` at the end of handleNanoRequest, following the same pattern as existing endpoints.

Important: The context parameter gains one new dependency — `generateCloudEmbeddings`. This is a function(texts, opts) that calls the Voyage API. It will be wired in playground.js (Task 2 handles that wiring).
