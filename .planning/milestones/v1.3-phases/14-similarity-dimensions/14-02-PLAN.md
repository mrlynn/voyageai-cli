**JS (add after nanoDoSimilarity function):**

Helper function nanoCosineSim(a, b): compute cosine similarity between two vectors. Since MRL truncation means lower dims are prefixes of higher dims, compare the first min(a.length, b.length) values. Implementation: dot product / (norm_a * norm_b). Handle zero-norm edge case (return 0).

window.nanoDoDimensions = async function():
1. hideError('nanoDimError')
2. Get text from nanoDimText, trim, validate non-empty
3. setLoading('nanoDimBtn', true)
4. fetch POST /api/nano/dimensions with { text }
5. On success, render:
   a. Stats: latency_ms
   b. For each dimension (256, 512, 1024, 2048), create a card showing:
      - Title: "{dim} dimensions" (with "Baseline" badge on 2048)
      - Stat rows: Norm (formatted to 4 decimals), Sparsity (formatted as percentage with 1 decimal), Vector length (the dim number)
      - Similarity preservation: compute cosine similarity between this dimension's embedding and the 2048 baseline embedding (use nanoCosineSim with the truncated prefix). For 2048 itself, show "1.0000 (baseline)". For others, show the score formatted to 4 decimals with a color indicator (green >= 0.99, yellow >= 0.95, red < 0.95).
   c. Render the 4 cards into nanoDimCards using the nano-dim-grid layout.
6. Show nanoDimResult with .visible class
7. Catch errors -> showError, finally -> setLoading false

Also update the nanoInit() function: after the lines that show/hide nanoSimilaritySection, add matching lines to show/hide nanoDimensionSection using the same pattern.

Use string concatenation (not template literals) for all HTML building.
