Create `src/lib/playground-nano-api.js` following the exact `playground-rag-api.js` pattern. Export a single `handleNanoRequest(req, res, context)` function that returns true if handled, false otherwise. Receive `readJsonBody`, `generateLocalEmbeddings`, and health check functions via the `context` parameter to keep the module testable and decoupled (per user decision).

**GET /api/nano/status (ENDP-01):**
- Call checkPython(), checkVenv(), checkModel(), checkDevice() from context
- Return: `{ ready: boolean, components: { python: {ok, version, hint?}, venv: {ok, path, hint?}, model: {ok, path, hint?}, bridge: {ok, hint?} } }`
- Top-level `ready` is true only when all components are ok
- Map health check `message` field to the appropriate detail field (version for python, path for venv/model)
- Include `hint` only when component is not ok
- Check fresh every call -- no caching (per user decision)

**POST /api/nano/embed (ENDP-02):**
- Parse JSON body: `{ text: string, dimension?: number, quantization?: string }`
- Input validation: text required, max 10000 chars. dimension must be one of [256, 512, 1024, 2048] if provided, default 1024. quantization must be one of ['float32', 'int8', 'uint8', 'binary'] if provided, default 'float32'. Return 400 with clear message on validation failure.
- Check nano readiness (call checkVenv and checkModel) -- if not ready, return 503 with `{ error: 'Nano bridge not ready', code: 'NANO_NOT_READY', hint: 'Run: vai nano setup' }`
- Call `generateLocalEmbeddings([text], { dimensions: dimension, precision: quantization })` from context
- Measure latency with `performance.now()`
- Return: `{ embedding: number[], dimension: number, quantization: string, model: 'voyage-4-nano', latency_ms: number }`

**POST /api/nano/similarity (ENDP-03):**
- Parse JSON body: `{ texts: string[], dimension?: number }`
- Input validation: texts required, must be array of 2-10 strings, each max 10000 chars. dimension defaults to 1024. Return 400 on violation.
- Check nano readiness -- 503 if not ready (same pattern as embed)
- Call `generateLocalEmbeddings(texts, { dimensions: dimension })` from context
- Compute NxN cosine similarity matrix using `cosineSimilarity` from `../lib/math.js`
- Measure latency
- Return: `{ texts: string[], matrix: number[][], dimension: number, latency_ms: number }`
- Texts echoed back for frontend label alignment (per user decision)

**POST /api/nano/dimensions (ENDP-04):**
- Parse JSON body: `{ text: string }`
- Input validation: text required, max 10000 chars. Return 400 on violation.
- Check nano readiness -- 503 if not ready
- For each dimension in [256, 512, 1024, 2048], call `generateLocalEmbeddings([text], { dimensions: dim })` from context
- Measure total latency
- Return: `{ text: string, dimensions: { 256: { embedding: number[], norm: number, sparsity: number }, 512: {...}, 1024: {...}, 2048: {...} }, latency_ms: number }`
- Compute norm as L2 norm (Math.sqrt of sum of squares), sparsity as fraction of near-zero values (Math.abs < 1e-6)

**Error handling pattern (per user decisions):**
- Wrap each endpoint body in try/catch
- On bridge errors: return 500 with `{ error: string, code: err.code || 'NANO_UNKNOWN' }`
- 503 for not-ready, 400 for validation, 500 for bridge failures
- Single attempt, no retry (per user decision)

**Module structure:** Use `'use strict'`, require only `math.js` at top level. Health checks and generateLocalEmbeddings come from context.
