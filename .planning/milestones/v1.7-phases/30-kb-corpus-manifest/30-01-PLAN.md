  <action>
1. Create the directory tree: `src/kb/corpus/` with subdirectories `explainers/`, `guides/`, `reference/`, `examples/`. Add `.gitkeep` files so empty dirs are tracked.

2. Create `scripts/generate-kb-manifest.js` -- a standalone Node.js script (no external deps beyond Node built-ins) that:
   - Recursively scans `src/kb/corpus/` for `.md` files
   - Parses YAML front matter from each file (implement simple YAML parser -- front matter is delimited by `---` lines, contains only `key: value` pairs for: title, type, section, difficulty)
   - Computes SHA-256 checksum of file content using `crypto.createHash('sha256')`
   - Derives `id` from filename without extension (e.g., `embeddings.md` -> `embeddings`)
   - Derives `path` as relative path from corpus root (e.g., `explainers/embeddings.md`)
   - Derives `url` as `https://docs.vaicli.com/kb/${path}` (placeholder URLs per user decision)
   - Estimates chunks using: `Math.ceil(wordCount / 380)` (approximating 512 tokens ~ 380 words, per existing chunker.js defaults of 512 tokens with 50 overlap)
   - Produces manifest.json with top-level fields:
     ```json
     {
       "version": "<read from package.json>",
       "generatedAt": "<ISO timestamp>",
       "chunkStrategy": "recursive",
       "chunkSize": 512,
       "chunkOverlap": 50,
       "embeddingModel": "voyage-4-large",
       "documents": [...]
     }
     ```
   - Each document entry: `{ id, path, url, title, type, section, difficulty, estimatedChunks, checksum }`
   - Writes output to `src/kb/corpus/manifest.json`
   - Script is executable: `#!/usr/bin/env node` with `'use strict';`
   - Prints summary to stdout: number of docs found per category, total estimated chunks

3. Make script executable: `chmod +x scripts/generate-kb-manifest.js`
  </action>

  </verify>
  <done>Directory tree exists at src/kb/corpus/ with 4 category subdirs. Generator script parses front matter, computes checksums, produces manifest.json with full spec schema.</done>

<task type="auto">
  <name>Task 2: Create release verification script</name>
  <files>scripts/verify-kb-manifest.js</files>
  <action>
Create `scripts/verify-kb-manifest.js` -- a standalone Node.js script that:

1. Reads `package.json` to get current version
2. Reads `src/kb/corpus/manifest.json` to get manifest version
3. Checks:
   - manifest.json exists
   - manifest.version matches package.json version exactly
   - Every file listed in manifest.documents actually exists on disk at `src/kb/corpus/{doc.path}`
   - Every `.md` file in `src/kb/corpus/` is listed in the manifest (no orphaned files)
   - All checksums are valid (recompute SHA-256 and compare)
4. Exit code 0 on success, exit code 1 on any failure
5. Prints clear pass/fail messages for each check using console.log (no color dependencies)
6. Script is executable: `#!/usr/bin/env node` with `'use strict';`

Example output:
```
[PASS] manifest.json exists
[PASS] Version match: 1.33.6
[PASS] All 30 manifest entries have matching files
[PASS] No orphaned files outside manifest
[PASS] All checksums valid
KB manifest verification: PASSED
```

Or on failure:
```
[FAIL] Version mismatch: manifest=1.33.5, package=1.33.6
[FAIL] Missing file: explainers/embeddings.md
KB manifest verification: FAILED
```

Make executable: `chmod +x scripts/verify-kb-manifest.js`
