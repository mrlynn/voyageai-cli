## Cross-Phase Integration

**Score: 11/11 wiring points connected**

All Phase 16 to Phase 17 data flows are properly wired:

- Config endpoint (`GET /api/chat/config`) returns detection booleans consumed by health dots and recommendation engine
- Embedding model selection persists through `POST /api/chat/config` to `.vai.json`
- One-click apply in welcome banner correctly sets provider, model, and embedding dropdowns then saves
- Both playground UI and CLI flag route embedding model through to `chatTurn` retrieval

### E2E Flows (3/3 complete)

1. **Playground onboarding flow** - Load -> detect services -> health dots -> recommend -> one-click apply -> config saved
2. **Playground embedding flow** - Select model -> send message -> backend validates -> correct embedder used
3. **CLI embedding flow** - `--embedding-model` flag -> resolution cascade -> validation -> correct embedder used
