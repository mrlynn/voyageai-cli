<task type="auto" tdd="true">
  <name>Task 1: Create TurnOrchestrator with event mapping and interrupt support</name>
  <files>src/lib/turn-orchestrator.js, test/lib/turn-orchestrator.test.js</files>
  <behavior>
    - TurnOrchestrator constructor accepts { sessionId, mode: 'pipeline'|'agent' }
    - Creates internal TurnStateMachine instance
    - `executePipelineTurn(generatorFn, args)` async generator: creates AbortController, transitions through states as generator yields events, re-yields events to caller
    - Event mapping for pipeline: start->VALIDATING->EMBEDDING, 'retrieval' event->RETRIEVING->RERANKING->BUILDING_PROMPT->GENERATING, 'chunk'->STREAMING, 'done'->PERSISTING->IDLE
    - `executeAgentTurn(generatorFn, args)` async generator: similar but maps 'tool_call'->TOOL_CALLING, 'chunk'->STREAMING
    - Both methods accept an AbortSignal in args; when signal fires, transitions to INTERRUPTED and returns partial response
    - `interrupt()` calls the internal AbortController.abort() and state machine interrupt()
    - `on(event, handler)` delegates to internal state machine for 'stateChange' events
    - Error during generator iteration calls sm.error() and does NOT rethrow (session continues)
    - Exposes getters: state, label, turnIndex (delegates to state machine)
    - Partial response preserved on interrupt: yields { type: 'interrupted', data: { partialResponse, metadata } }
  </behavior>
  <action>
    Create `src/lib/turn-orchestrator.js` as CommonJS module ('use strict').

    The TurnOrchestrator class:

    1. Constructor stores sessionId, mode, creates TurnStateMachine({ sessionId, mode })
    2. `async *executePipelineTurn(params)` where params = { query, db, collection, llm, history, opts }:
       - Creates AbortController, stores as this._controller
       - Passes signal to opts: `opts.signal = this._controller.signal`
       - Transitions: IDLE->VALIDATING->EMBEDDING (before calling generator)
       - Calls chatTurn(params) and iterates its events
       - On 'retrieval': transition EMBEDDING->RETRIEVING (if retrieval has docs), then yield the event
       - On 'history': yield the event (no state change)
       - On first 'chunk': transition to GENERATING then STREAMING, yield chunk
       - On subsequent 'chunk': yield chunk (stay in STREAMING)
       - On 'done': transition STREAMING->PERSISTING->IDLE, yield done
       - Wrap iteration in try/catch: on AbortError or signal.aborted, transition to INTERRUPTED, yield interrupted event with partialResponse
       - On other errors: call sm.error(err), yield { type: 'error', data: { message: err.message, state: sm.state } }, do NOT throw
    3. `async *executeAgentTurn(params)` where params = { query, llm, history, opts }:
       - Similar structure but different state path
       - IDLE->VALIDATING->GENERATING (agent skips embedding/retrieval)
       - On 'tool_call': transition to TOOL_CALLING, yield event, transition back to GENERATING
       - On 'chunk': transition to STREAMING (if not already), yield chunk
       - On 'done': STREAMING->PERSISTING->IDLE
    4. `interrupt()`: calls this._controller?.abort(), calls sm.interrupt()
    5. Delegate getters and event methods to internal sm
    6. `getStateMachine()` returns the internal sm for direct access if needed

    For the test file, use mock generators (async function*) that yield controlled events. Test:
    - Pipeline turn progresses through correct states
    - Agent turn progresses through correct states with tool_call
    - stateChange events fire with correct from/to for each transition
    - Error in generator triggers error recovery (state returns to IDLE)
    - Interrupt during streaming yields interrupted event with partial response

    NOTE: The existing chatTurn/agentChatTurn generators do not currently accept AbortSignal. The orchestrator wraps them, so interrupt works by: (a) setting a flag that prevents further iteration, and (b) calling sm.interrupt(). The actual AbortSignal propagation into the LLM stream is a best-effort enhancement -- if the generator doesn't support it, the orchestrator catches the abort and handles cleanup. Do NOT modify chat.js generators in this plan.
  </action>

  </verify>
  <done>TurnOrchestrator correctly maps generator events to state transitions for both pipeline and agent modes, handles interrupts and errors gracefully, and all tests pass</done>
</task>
