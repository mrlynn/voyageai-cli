<feature>
  <name>TurnStateMachine + Token Estimator</name>
  <files>src/lib/turn-state.js, test/lib/turn-state.test.js</files>
  <behavior>
    ## State Machine Core (SM-01, SM-02)

    - STATES enum contains: IDLE, VALIDATING, EMBEDDING, RETRIEVING, RERANKING, BUILDING_PROMPT, GENERATING, STREAMING, PERSISTING, INTERRUPTED, ERROR_TURN, TOOL_CALLING
    - Constructor accepts { sessionId, mode } where mode is 'pipeline' or 'agent'
    - Initial state is IDLE
    - `transition(targetState, metadata)` moves to valid target and returns the machine
    - Pipeline valid transitions: IDLE->VALIDATING, VALIDATING->EMBEDDING, EMBEDDING->RETRIEVING, RETRIEVING->RERANKING, RERANKING->BUILDING_PROMPT, BUILDING_PROMPT->GENERATING, GENERATING->STREAMING, STREAMING->PERSISTING, PERSISTING->IDLE
    - Agent valid transitions include: GENERATING->TOOL_CALLING, TOOL_CALLING->GENERATING (loop), plus IDLE->VALIDATING, VALIDATING->GENERATING, GENERATING->STREAMING, STREAMING->PERSISTING, PERSISTING->IDLE
    - Any state can transition to INTERRUPTED or ERROR_TURN (except IDLE)
    - INTERRUPTED->IDLE is valid
    - ERROR_TURN->IDLE is valid (auto-recovery, SM-06)
    - Invalid transitions throw Error with message containing both the current state name and the target state name
    - `state` getter returns current state string

    ## Events (SM-03)

    - Each successful transition emits 'stateChange' event
    - Event payload: { from: string, to: string, sessionId: string, turnIndex: number, timestamp: number, durationMs: number, metadata: object }
    - durationMs is time spent in the `from` state (difference between current timestamp and when we entered `from`)
    - turnIndex starts at 0, increments when transitioning from IDLE (start of new turn)

    ## Labels (SM-04)

    - LABELS is a static map from state name to human-readable string
    - LABELS.IDLE = 'Ready', LABELS.VALIDATING = 'Validating...', LABELS.EMBEDDING = 'Embedding...', LABELS.RETRIEVING = 'Searching...', LABELS.RERANKING = 'Reranking...', LABELS.BUILDING_PROMPT = 'Building prompt...', LABELS.GENERATING = 'Generating...', LABELS.STREAMING = 'Streaming...', LABELS.PERSISTING = 'Saving...', LABELS.INTERRUPTED = 'Interrupted', LABELS.ERROR_TURN = 'Error', LABELS.TOOL_CALLING = 'Calling tools...'
    - `label` getter returns LABELS[currentState]

    ## Error Recovery (SM-06)

    - `error(err, metadata)` method transitions to ERROR_TURN then automatically to IDLE
    - Two stateChange events emitted: one for X->ERROR_TURN, one for ERROR_TURN->IDLE
    - The error message is included in the ERROR_TURN event metadata as `error`

    ## Token Estimator (MEM-06)

    - `estimateTokens(text)` returns Math.ceil(text.length / 4)
    - Returns 0 for empty string, null, undefined
    - Works with any string input

    ## Interrupt (SM-05 - state machine part)

    - `interrupt(metadata)` method transitions current state to INTERRUPTED
    - Throws if already in IDLE (nothing to interrupt)
    - metadata can include `partialResponse` for later retrieval
  </behavior>
  <implementation>
    Create src/lib/turn-state.js as a CommonJS module (per project convention -- 'use strict' at top).

    1. Define STATES as a frozen object with string values matching the key names
    2. Define LABELS as a frozen object mapping state names to display strings
    3. Define TRANSITIONS as a Map<state, Set<state>> encoding all valid transitions (pipeline + agent combined, since both modes share the same machine; the orchestrator in Plan 02 decides which path to take)
    4. Add universal transitions: any non-IDLE state can go to INTERRUPTED or ERROR_TURN
    5. TurnStateMachine extends require('node:events').EventEmitter
    6. Constructor stores sessionId, mode, initializes _state=IDLE, _turnIndex=0, _stateEnteredAt=Date.now()
    7. transition(target, metadata={}) validates, updates state, emits event, returns this
    8. error(err, metadata={}) calls transition(ERROR_TURN, {error: err.message||err, ...metadata}) then transition(IDLE)
    9. interrupt(metadata={}) validates not IDLE, calls transition(INTERRUPTED, metadata)
    10. Getters: state, label, turnIndex
    11. estimateTokens as standalone exported function

    Module exports: { TurnStateMachine, STATES, LABELS, estimateTokens }
  </implementation>
