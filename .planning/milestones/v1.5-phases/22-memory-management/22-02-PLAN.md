  <files>src/lib/memory-summarizer.js, test/lib/memory-summarizer.test.js</files>
  <behavior>
    - summarizeTurns(turns, llm) calls llm.chat() with a system prompt asking for a concise summary of the conversation turns, returns the summary string
    - SummarizationStrategy.select({ turns, budgetTokens, llm, threshold }) splits turns into old (to summarize) and recent (verbatim)
    - Split point: work backwards from newest, accumulate tokens until 60% of budget is used for recent turns. Remaining turns are "old"
    - If old turns exist and total token estimate exceeds threshold (default 80%) of budget: summarize old turns into a single summary message { role: 'system', content: '[Summary of earlier conversation]...' }
    - If below threshold: fall back to SlidingWindowStrategy behavior (no summarization needed)
    - Returns array: [summary_message?, ...recent_turns]
    - When llm is null or summarization fails, falls back to SlidingWindowStrategy
    - summarizeTurns formats turns as "User: ...\nAssistant: ..." pairs for the LLM
  </behavior>

  <behavior>
    CrossSessionRecall:
    - Constructor accepts { summaryStore, embedFn, embeddingModel, topK }
    - recall(query, currentSessionId) embeds the query with inputType='query' using the configured model, then runs a $vectorSearch aggregation on vai_session_summaries to find the topK most relevant past session summaries, excluding the current session
    - Returns Array<{ sessionId, summary, score }> sorted by relevance
    - If summaryStore is not connected or embedFn fails, returns empty array (graceful degradation)
    - Default embeddingModel: 'voyage-4-lite' for queries (asymmetric: voyage-4-large for indexing is done at summary storage time)

    HierarchicalStrategy:
    - static async select({ turns, budgetTokens, llm, recall, currentSessionId, query })
    - Allocates budget: 20% for cross-session recall, 20% for summaries, 60% for recent verbatim turns
    - Step 1: If recall provided, get cross-session context and format as system message
    - Step 2: Split turns into old and recent (like SummarizationStrategy)
    - Step 3: If old turns exist, summarize them
    - Step 4: Assemble: [cross_session_context?, summary?, ...recent_turns]
    - Falls back gracefully: no recall → skip step 1, no llm → skip step 3
    - Register as 'hierarchical' strategy in MemoryManager
  </behavior>
