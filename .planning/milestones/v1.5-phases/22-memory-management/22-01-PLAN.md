  <name>Task 1: TDD MemoryBudget — token budget allocation</name>
  <files>src/lib/memory-budget.js, test/lib/memory-budget.test.js</files>
  <behavior>
    - MemoryBudget constructor accepts { modelLimit, reservedResponse } with defaults { modelLimit: 128000, reservedResponse: 4096 }
    - computeHistoryBudget({ systemPromptTokens, contextTokens, currentMessageTokens }) returns the remaining tokens for history = modelLimit - reservedResponse - systemPromptTokens - contextTokens - currentMessageTokens
    - If the computed budget is negative, returns 0 (never negative)
    - estimateSlotTokens({ systemPrompt, contextDocs, currentMessage }) uses estimateTokens to compute each slot's tokens, then calls computeHistoryBudget
    - Edge: empty systemPrompt → 0 tokens; null contextDocs → 0 tokens
    - Edge: modelLimit of 4096 with large prompt → returns 0 or very small budget
    - getBreakdown() returns an object with { modelLimit, reservedResponse, systemPrompt, context, currentMessage, historyBudget } after calling estimateSlotTokens
  </behavior>
  <action>

    2. computeHistoryBudget with known values returns correct remainder
    3. computeHistoryBudget returns 0 when sum exceeds modelLimit
    4. estimateSlotTokens with real strings computes correct token estimates
    5. estimateSlotTokens with null/empty inputs returns full available budget
    6. getBreakdown returns all slot sizes after computation
    7. Small modelLimit (4096) with large prompt returns 0 budget

    GREEN: Create src/lib/memory-budget.js:

    - getBreakdown() → object with all slot sizes

<task type="auto" tdd="true">
  <name>Task 2: TDD MemoryManager + SlidingWindowStrategy</name>
  <files>src/lib/memory-strategy.js, test/lib/memory-strategy.test.js</files>
  <behavior>
    - SlidingWindowStrategy.select(turns, budgetTokens) returns the most recent turns that fit within budgetTokens
    - Turns are { role, content } objects; token cost = estimateTokens(turn.content)
    - Works backwards from most recent turn, accumulating token cost
    - If the very first (most recent) turn exceeds budget, it is still included (always at least 1 turn if turns exist)
    - Empty turns array → returns []
    - Zero budget → returns [] (or single most recent if turns exist — design choice: return [])
    - MemoryManager constructor accepts { strategies } map, defaultStrategy name
    - MemoryManager.buildHistory({ turns, budget, strategy? }) selects strategy by name and calls select()
    - MemoryManager.getStrategyNames() returns available strategy names
    - Default strategies registered: 'sliding_window'
    - Unknown strategy name throws Error
  </behavior>
  <action>
    RED: Write test/lib/memory-strategy.test.js with node:test. Test cases:

    3. SlidingWindowStrategy.select with 0 budget → returns empty array
    4. SlidingWindowStrategy.select with empty turns → returns empty array
    5. SlidingWindowStrategy.select preserves order (oldest first in returned array)
    6. MemoryManager.buildHistory uses sliding_window by default
    7. MemoryManager.buildHistory with explicit strategy name
    8. MemoryManager.getStrategyNames returns ['sliding_window']
    9. MemoryManager.buildHistory with unknown strategy throws
