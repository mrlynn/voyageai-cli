2. **Archive flow — generate and store session summary:**

   In the `/archive` slash command handler (around line 1084), after `sessionStore.transitionLifecycle(sessionId, 'archived')`, add summary generation:

   ```javascript
   // Generate session summary for cross-session recall
   try {
     const { summarizeTurns } = require('../lib/memory-summarizer');
     const { SessionSummaryStore } = require('../lib/session-summary-store');
     const { generateEmbeddings } = require('../lib/api');

     const allTurns = history.getMessages();
     if (allTurns.length >= 2) {
       // 1. Generate summary via LLM
       const summary = await summarizeTurns(allTurns, llm);
       if (summary) {
         // 2. Embed summary for vector search (use voyage-4-large for document embedding — asymmetric)
         const summaryStore = new SessionSummaryStore({ db: db || 'vai' });
         try {
           const embedResult = await generateEmbeddings([summary], {
             model: 'voyage-4-large',
             inputType: 'document',
           });
           const embedding = embedResult.data[0].embedding;

           // 3. Store in vai_session_summaries
           await summaryStore.storeSummary({ sessionId, summary, embedding });
           if (!opts.quiet && !opts.json) {
             console.log(pc.dim('  Session summary saved for cross-session recall.'));
           }
         } finally {
           await summaryStore.close();
         }
       }
     }
   } catch {
     // Summary generation is non-fatal — session is already archived
   }
   ```

   IMPORTANT: The `llm` variable is available in the slash command handler context — it's the LLM provider instance from `createLLMProvider(opts)`. Pass it through the handleSlashCommand ctx object. Currently ctx has: `{ history, opts, db, collection, llm, rl, ... }` — `llm` is already there.

3. **Resume flow — initialize CrossSessionRecall:**

   In the session resume block (around line 303), after loading turns for a resumed session, initialize CrossSessionRecall and attach it to the memoryManager:

   ```javascript
   // After session resume turn loading (around line 312-319)
   // Initialize cross-session recall for resumed sessions
   let crossSessionRecall = null;
   if (sessionStore && !sessionStore.isFallbackMode) {
     try {
       const { SessionSummaryStore } = require('../lib/session-summary-store');
       const { CrossSessionRecall } = require('../lib/cross-session-recall');
       const { generateEmbeddings } = require('../lib/api');

       const summaryStore = new SessionSummaryStore({ db: db || 'vai' });
       await summaryStore._connect();
       crossSessionRecall = new CrossSessionRecall({
         summaryStore,
         embedFn: generateEmbeddings,
       });
     } catch {
       // Cross-session recall init failure is non-fatal
     }
   }
   ```

   Then when creating the memoryManager (from Plan 24-01), pass the recall instance:
   ```javascript

     memoryManager.setOpts({ recall: crossSessionRecall, currentSessionId: sessionId });
   }
   ```

   This ensures the hierarchical strategy has the current query for cross-session recall vector search.
