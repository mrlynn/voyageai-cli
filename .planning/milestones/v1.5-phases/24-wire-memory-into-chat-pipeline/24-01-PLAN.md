**In src/lib/chat.js:**

1. Add imports at top:


2. Modify `chatTurn()` signature to accept `memoryManager` and `memoryStrategy` in opts:
   - `opts.memoryManager` — MemoryManager instance (optional, falls back to getMessagesWithBudget)
   - `opts.memoryStrategy` — strategy name string (optional, defaults to manager's default)
   - `opts.llm` — LLM instance (needed for summarization/hierarchical strategies)

3. Replace the hardcoded budget block (lines 251-252) with:
   ```javascript
   let historyMessages;
   if (opts.memoryManager) {
     const budget = new MemoryBudget();
     const historyBudget = budget.estimateSlotTokens({
       systemPrompt: opts.systemPrompt || '',
       contextDocs: docs,
       currentMessage: query,
     });
     const allTurns = history.getMessages();
     historyMessages = await opts.memoryManager.buildHistory({
       turns: allTurns,
       budget: historyBudget,
       strategy: opts.memoryStrategy,
     });
   } else {
     // Legacy fallback
     const historyBudget = opts.historyBudget || 4000;
     historyMessages = history.getMessagesWithBudget(historyBudget);
   }
   ```

   IMPORTANT: The `buildHistory` call for summarization and hierarchical strategies is async (they call LLM or vector search). MemoryManager.buildHistory currently calls `strat.select()` synchronously. Update MemoryManager.buildHistory to `await strat.select()` — the sliding_window strategy returns synchronously but `await` on a non-Promise is harmless. Mark `buildHistory` as `async`.

4. Apply the same pattern in `agentChatTurn()` (line 347), replacing `history.getMessagesWithBudget(8000)` with the same MemoryBudget + MemoryManager approach. For agent mode, pass a larger reservedResponse (8192) since agents use more output tokens.

**In src/lib/memory-strategy.js:**

5. Make `MemoryManager.buildHistory()` async:
   ```javascript
   async buildHistory({ turns, budget, strategy } = {}) {
     const name = strategy || this._defaultStrategy;
     const strat = this._strategies.get(name);
     if (!strat) throw new Error(`Unknown strategy: ${name}`);
     return await strat.select(turns, budget);
   }
   ```
   Note: SlidingWindowStrategy.select returns synchronously, but `await` on a non-Promise just resolves it. SummarizationStrategy.select and HierarchicalStrategy.select are already async.

   IMPORTANT: SlidingWindowStrategy.select(turns, budgetTokens) takes positional args, but SummarizationStrategy.select and HierarchicalStrategy.select take an options object `{ turns, budgetTokens, llm, ... }`. The MemoryManager.buildHistory must normalize the call signature. Update buildHistory to pass an options object to all strategies, and update SlidingWindowStrategy.select to also accept an options object form:

   ```javascript
   static select(turnsOrOpts, budgetTokens) {
     // Support both: select(turns, budget) and select({ turns, budgetTokens })
     let turns, budget;
     if (Array.isArray(turnsOrOpts)) {
       turns = turnsOrOpts;
       budget = budgetTokens;
     } else {
       turns = turnsOrOpts.turns;
       budget = turnsOrOpts.budgetTokens;
     }
     // ... existing logic with turns, budget
   }
   ```

   Then MemoryManager.buildHistory always passes the options object:
   ```javascript
   return await strat.select({ turns, budgetTokens: budget, ...this._extraOpts });
   ```

   Add `_extraOpts` to constructor so callers can set `llm`, `recall`, `currentSessionId` etc. once:
   ```javascript
   constructor({ strategies, defaultStrategy, llm, recall, currentSessionId, query } = {}) {
     // ... existing
     this._extraOpts = {};
     if (llm) this._extraOpts.llm = llm;
     if (recall) this._extraOpts.recall = recall;
     if (currentSessionId) this._extraOpts.currentSessionId = currentSessionId;
   }
   setOpts(opts) { Object.assign(this._extraOpts, opts); }
   ```
