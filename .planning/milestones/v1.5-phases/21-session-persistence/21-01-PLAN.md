<feature>
  <name>SessionStore with MongoDB CRUD, lifecycle, TTL, and fallback</name>
  <files>src/lib/session-store.js, test/lib/session-store.test.js</files>
  <behavior>
    ## SESSION_STATES enum (SM-07)

    - SESSION_STATES = { INITIALIZING: 'initializing', ACTIVE: 'active', PAUSED: 'paused', ARCHIVED: 'archived' }
    - Valid transitions: INITIALIZING->ACTIVE, ACTIVE->PAUSED, PAUSED->ACTIVE, ACTIVE->ARCHIVED, PAUSED->ARCHIVED
    - Invalid transitions throw Error with descriptive message

    ## SessionStore class

    Constructor: `new SessionStore({ db, ttlDays })` where db is the database name (string), ttlDays defaults to 90.
    - Does NOT connect in constructor -- connection is lazy on first operation
    - Stores db name and ttlDays for later use

    ### Session CRUD (SES-01)

    - `createSession({ model, provider, mode })` -> returns session object: `{ _id: UUID, model, provider, mode, lifecycleState: 'initializing', createdAt: Date, updatedAt: Date }`
    - Inserts into `vai_sessions` collection
    - `getSession(sessionId)` -> returns session object or null
    - `updateSession(sessionId, updates)` -> updates `updatedAt` plus any allowed fields; returns updated doc
    - `transitionLifecycle(sessionId, targetState)` -> validates transition is legal, updates lifecycleState and updatedAt; throws on invalid transition (SM-07)

    ### Turn CRUD (SES-02)

    - `storeTurn({ sessionId, turnIndex, role, content, context, tokens, timing })` -> inserts into `vai_chat_turns` with all fields plus `createdAt: Date`
    - `tokens` object: `{ embed, rerank, llmInput, llmOutput, estimated }` (estimated via estimateTokens for the content)
    - `timing` object: `{ startedAt, completedAt, durationMs }`
    - `getTurns(sessionId, { limit, offset } = {})` -> returns turns sorted by turnIndex ascending
    - `getLatestTurns(sessionId, count)` -> returns last N turns (for history window)
    - Compound index on `{ sessionId: 1, turnIndex: 1 }` with unique constraint

    ### TTL (SES-05)

    - `ensureIndexes()` creates:
      1. Compound index on vai_chat_turns: `{ sessionId: 1, turnIndex: 1 }` unique
      2. TTL index on vai_chat_turns: `{ createdAt: 1 }` with `expireAfterSeconds: ttlDays * 86400`
      3. Index on vai_sessions: `{ lifecycleState: 1, updatedAt: -1 }` for listing
    - Called lazily on first write operation (idempotent)
    - Index creation failures are non-fatal (logged but not thrown)

    ### Graceful Fallback (SES-06)

    - `_getCollection(name)` internal method: tries getMongoCollection, on failure sets `this._fallbackMode = true` and returns null
    - When `_fallbackMode` is true, all operations use in-memory Maps:
      - `this._memSessions = new Map()` for sessions
      - `this._memTurns = new Map()` for turns (keyed by sessionId, value is array)
    - `isFallbackMode` getter returns current mode
    - Every public method has try/catch: if MongoDB throws mid-operation, switches to fallback mode and retries with in-memory store
    - No error messages printed to console -- caller decides how to inform user

    ## Test behavior

    - Tests use mock MongoDB collections (not real DB)
    - Mock collection: object with `insertOne`, `findOne`, `find().sort().limit().toArray()`, `updateOne`, `createIndex` methods
    - Session creation returns proper document shape
    - Turn storage and retrieval maintain order
    - Lifecycle transitions: valid paths succeed, invalid paths throw
    - TTL: ensureIndexes calls createIndex with correct expireAfterSeconds
    - Fallback: when mock throws, store switches to in-memory and operations still succeed
    - Fallback: after fallback mode is active, all subsequent operations use in-memory (no retry of MongoDB)
  </behavior>
  <implementation>
    Create `src/lib/session-store.js` as CommonJS module ('use strict').

    1. Define SESSION_STATES as a frozen object: `{ INITIALIZING: 'initializing', ACTIVE: 'active', PAUSED: 'paused', ARCHIVED: 'archived' }`

    2. Define LIFECYCLE_TRANSITIONS as a Map:
       ```
       initializing -> Set([active])
       active -> Set([paused, archived])
       paused -> Set([active, archived])
       ```

    3. SessionStore class:
       - Constructor: stores `this._db = db`, `this._ttlDays = ttlDays || 90`, `this._fallbackMode = false`, `this._indexesEnsured = false`, `this._memSessions = new Map()`, `this._memTurns = new Map()`, `this._client = null`, `this._sessionsCol = null`, `this._turnsCol = null`
       - `async _connect()`: if already connected, return. Call getMongoCollection twice (for vai_sessions and vai_chat_turns). Store client and collections. On error, set `_fallbackMode = true`.
       - `async _ensureIndexes()`: if `_indexesEnsured` or `_fallbackMode`, return. Create the 3 indexes. Set `_indexesEnsured = true`. Errors are caught and ignored.
       - `async close()`: if `_client`, call `_client.close()`. Reset connection state.

    4. Session methods:
       - `createSession({ model, provider, mode })`: generate UUID via crypto.randomUUID(), build doc, insert into collection (or Map in fallback), transition to ACTIVE immediately, return doc
       - `getSession(id)`: findOne by _id (or Map.get in fallback)
       - `updateSession(id, updates)`: updateOne with $set (or Map merge in fallback)
       - `transitionLifecycle(id, target)`: get current session, validate transition, updateSession with new state

    5. Turn methods:
       - `storeTurn(turnData)`: auto-add estimateTokens(content) to tokens.estimated, insert into collection (or push to Map array in fallback), ensure indexes on first write
       - `getTurns(sessionId, opts)`: find by sessionId, sort by turnIndex, apply limit/offset
       - `getLatestTurns(sessionId, count)`: find by sessionId, sort by turnIndex desc, limit count, reverse result

    6. Export: `{ SessionStore, SESSION_STATES }`

    For tests: create a `mockCollection()` helper that returns an object with jest-like stubs. Use node:test `mock.fn()` for method tracking. Test each method in isolation with the mock, then test fallback by making the mock throw.
